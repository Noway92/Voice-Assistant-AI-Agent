MINIMAL INFERENCE CODE
--------------------------------------------------------------------------------
```python
from TTS.tts.configs.xtts_config import XttsConfig
from TTS.tts.models.xtts import Xtts
import torch

# Load model
config = XttsConfig()
config.load_json("config.json")
model = Xtts.init_from_config(config)
model.load_checkpoint(config, checkpoint_path="best_model.pth")
model.cuda()  # Use GPU (or model.cpu() for CPU inference)

# Generate speech
outputs = model.synthesize(
    text="Your text here",
    config=config,
    speaker_wav="speaker_reference.wav",  # Your voice sample
    language="en",  # "en", "fr", "es", "de", "ja", etc.
    temperature=0.6,
    repetition_penalty=15.0,
    top_k=15,
    top_p=0.725
)

# Save audio
import torchaudio
torchaudio.save("output.wav", torch.tensor(outputs["wav"]).unsqueeze(0), 24000)
```

OPTIMAL INFERENCE PARAMETERS
--------------------------------------------------------------------------------
Moderate (Recommended):
- temperature: 0.6
- repetition_penalty: 15.0
- top_k: 15
- top_p: 0.725
- length_penalty: 2.0 (ignored with num_beams=1, but can set)

Conservative (More stable, less variation):
- temperature: 0.3
- repetition_penalty: 10.0
- top_k: 50
- top_p: 0.8

Aggressive (NOT recommended - increases hallucination):
- temperature: 0.7-0.85
- repetition_penalty: 30.0
- top_k: 10
- top_p: 0.6

SPEAKER REFERENCE AUDIO
--------------------------------------------------------------------------------
What it is:
- A short (3-10 second) sample of the voice you want to clone
- One of your training audio files from dataset/wavs/
- Example: "New Recording 1.wav"

What it does:
- XTTS analyzes this at inference time to extract voice characteristics
- Must be provided every time you generate speech
- Acts as "confirmation" for your fine-tuned model

Why training matters:
- Pretrained XTTS: Can clone any voice but with mediocre quality
- Your fine-tuned model: Specialized for YOUR voice with high fidelity
- Reference audio + fine-tuned model = Best quality

Best practices:
- Use clean, clear recording (no background noise)
- 3-6 seconds is ideal
- Natural speaking style
- Can be any sentence (doesn't need to relate to generated text)

WHY CONFIG.JSON IS NEEDED
--------------------------------------------------------------------------------
Contains two types of information:

Model Architecture (REQUIRED for inference):
- num_chars, gpt_num_audio_tokens
- gpt_layers, gpt_n_model_channels
- Audio processing settings (sample rates, mel specs)
- Tokenizer configuration

Training Settings (IGNORED during inference):
- batch_size, grad_accum_steps
- optimizer, lr_scheduler
- save_step, print_step

Why needed: TTS library uses config.json to reconstruct the model with the
exact architecture it was trained with, then loads your weights into it.
Without it, the library wouldn't know the model's shape/structure.

GPU INFERENCE TROUBLESHOOTING
--------------------------------------------------------------------------------
If you get CUDA/memory errors during inference:

1. Clear cache before generating:
```python
import torch
torch.cuda.empty_cache()
```

2. Use CPU if needed (slower but reliable):
```python
model.cpu()  # Instead of model.cuda()
```

3. Generate shorter texts (long text uses more VRAM)

Your training proves GPU works - if inference fails, it's just memory management.
6GB VRAM is tight for large batches, but single inference should work fine.
